# Contributing

Если хотите помочь с проектом:

## Сообщить о баге или проблеме

1. Проверьте, нет ли уже похожего [Issue](../../issues)
2. Создайте новый Issue с описанием проблемы
3. Приложите логи или скриншоты, если есть

## Предложить улучшение

1. Создайте Feature Request через [Issues](../../issues/new/choose)
2. Опишите, что хотите добавить и зачем это нужно

## Если хотите написать код

### Настройка окружения

```bash
# Fork репозитория и клонируйте
git clone https://github.com/YOUR_USERNAME/Yeb-Ich.git
cd Yeb-Ich

# Установите зависимости (если есть)
# npm install или pip install -r requirements.txt
```

## Процесс разработки

```bash
# Создайте ветку
git checkout -b fix/bug-name
# или
git checkout -b feature/feature-name

# Внесите изменения, закоммитьте
git add .
git commit -m "fix: описание"

# Отправьте
git push origin fix/bug-name
```

## Стиль кода

- Читаемый код
- Понятные имена переменных
- Комментарии для сложных мест

Коммиты:
```
fix: краткое описание
feat: краткое описание
docs: краткое описание
```

## Pull Request

1. Запушьте ветку: `git push origin your-branch`
2. Создайте Pull Request на GitHub
3. Опишите что изменено и зачем
4. Дождитесь ревью

## Вопросы по безопасности

**НЕ создавайте публичные Issues для уязвимостей.**

Смотрите [SECURITY.md](SECURITY.md) - там указано, куда писать.
